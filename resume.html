The main goal of this project is to centralize specific production and operational data from Ge/Ga generators into a SQL Server database. These data originate from different types of files (Excel, CSV, or PDF) or from manual entry (currently done in Excel files).

The R&D team also wanted the ability to directly input various data into the database with great flexibility, meaning they could display, insert, modify, or delete data. This was all aimed at implementing a reporting system with the database to generate reports or charts from the centralized data.

Initially, the project was supposed to be done entirely in Python, both frontend and backend. After careful consideration with the team, it was decided to develop a full-stack project, meaning a web application, a Python application for the backend, and the SQL Server database.

The web application provides a more user-friendly and visually appealing interface as requested by the R&D team, making the frontend aspect easier. The Python application will be used to process the various files and send them to the database.

After completing my internship, I was hired by IRE as an employee to continue and finalize the 'Ge-Ga Project,' allowing me to see it through to completion and ensure its implementation aligned with the R&D team's expectations.
